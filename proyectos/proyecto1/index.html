App de Citas Médicas

Gestión de citas médicas para pacientes y médicos.

Introducción

Esta aplicación permite a pacientes y médicos gestionar citas, documentos médicos, medicación e información personal de forma sencilla y eficiente.

Tecnologías utilizadas

  • Frontend: Angular, Angular Material, Tailwind CSS
  • Backend: Spring Boot, Spring Security, JWT
  • Base de datos: MongoDB
  • Control de versiones: GitHub

Arquitectura

  • SPA Angular (Router + Guards) → API REST Spring Boot (MVC) → MongoDB (colecciones y GridFS)
  • Autenticación con JWT en Authorization: Bearer ...; CORS habilitado por dominio
  • Auditoría: derivaciones, descargas de documentos y cambios de estado de citas

API principal

  • Auth: POST /api/auth/login, GET /api/auth/me
  • Citas: GET/POST/PUT/DELETE /api/citas, GET /api/citas?medicoId|pacienteId&fecha
  • Derivaciones: POST /api/derivaciones
  • Especialistas: GET /api/especialistas, GET /api/especialistas/{id}/disponibilidad
  • Documentos: POST/GET /api/documentos, GET /api/documentos/{id}/descargar
  • Pacientes: GET /api/pacientes?dni|nombre

Modelo de datos (resumen)

  • pacientes: { id, dni, nombre, medicosAsignados[], informacionMedica{}, medicación[], documentos[] }
  • medicos: { id, dni, especialidad, horario, notificaciones[] }
  • citas: { id, pacienteId, medicoId, fechaHora, estado, tipoCita }
  • documentos (GridFS): metadatos { pacienteId, medicoId, tipo, titulo, version }
  • secretarias, citasInvitado

Estados y reglas clave

  • Cita: PROGRAMADA → COMPLETADA | CANCELADA (por rol y por tiempo)
  • Solapes: no se permiten citas/derivaciones si hay conflicto horario (409)
  • Archivos: PDF hasta 10 MB, validación MIME y nombre
