Subir Documentos

Formulario para que los médicos suban documentos clínicos de los pacientes.

Descripción

Los médicos pueden buscar un paciente, seleccionar un archivo PDF, agregar un título y descripción, y subir el documento al sistema.

Capturas de pantalla

Formulario para subir documentos clínicos.

Rutas relacionadas

  • Frontend: `/subir-documentos` (Angular Router)
  • Backend: `/api/documentos` (Spring Boot)

Validaciones

  • Archivo PDF, tamaño ≤ 10 MB, nombre sin caracteres peligrosos
  • Título 3–120 caracteres; descripción 0–500

Metadatos

  • GridFS metadata: pacienteId, medicoId, titulo, tipo, version, hash

API y errores

  • POST /api/documentos (multipart form-data: file, pacienteId, titulo, descripcion, tipo)
  • Respuestas: 201 creado | 415 tipo no permitido | 413 demasiado grande | 422 inválido

Ejemplo multipart

file: informe.pdf
pacienteId: p1
titulo: Informe radiología
descripcion: Control trimestral
tipo: INFORME
